# MicMeas / OtoScope  `v0.0.5`

ブラウザ完結の「音の定点点検ノート」（PoC）。マイクで測って、埋め込みで観測し、ラベルを貯めて **自分専用の判定モデルを育てる** — すべて端末内で。

- 録音 → **端末内**で埋め込み計算（YAMNet / DSP / MelCNN）
- IndexedDB に蓄積・可視化・簡易定点観測
- 正常/異常ラベルから **クライアントサイドで判定モデルを学習**し、次の測定を自動判定
- モデルも同梱配信のため **実行時の外部通信はゼロ**（閉域LANでも動作）
- GitHub Pages（HTTPS）向け静的アプリ・PWA対応

**公開 URL:** https://smyrk2031.github.io/MicMeas/

設計思想・パイプライン・弱点・今後の方向性は [`docs/DESIGN.md`](docs/DESIGN.md) 参照。

## コンセプト

「毎日の点検で“いつもと違う”を捕まえ、ラベルを貯めて判定を賢くする、ローカル完結の点検ノート」。判定AIを作り込むのではなく、**埋め込んで・並べて・人が解釈し・ラベルで育てる** human-in-the-loop の状態監視ツールです。

## 埋め込み空間

| 空間 | 次元 | 性質 |
|---|---|---|
| **YAMNet** | 1024 | 意味的。AudioSet学習済みCNN。521音クラス判定も同時 |
| **DSP** | 80 | 物理的。40帯域 log-mel の平均＋標準偏差 |
| **MelCNN（実験）** | 512 | メル画像を固定重み(学習なし)の軽量CNNで非線形変換。シード固定で端末間再現。将来は学習済み重みに差し替え可能 |

## 画面

- **🎙 測定**: 大きな録音ボタン＋**判定バナー（信号カラー）**＋最近の推移トレンド。保存モデルがあれば正常/異常を即判定
- **📊 レコード解析**
  - **🎨 可視化**: カラーマップ（音紋の縦積み）＋埋め込みマップ（PCA）を同時表示・相互に選択連動。正常●/異常◆/未ラベル○
  - **🧪 評価**: 正常/異常ラベルで線形プローブ（ロジスティック回帰）を学習し、交差検証で AUC・F1・混同行列・誤判定一覧を表示。埋め込み空間を比較
  - **🧩 モデル**: 学習済み判定器の一覧・新規作成・編集/再学習・有効化・全レコード再判定
  - **📋 レコード一覧**: テーブル
- **詳細モーダル**: 再生・帯域ミュート・スペクトログラム分析・ラベル付け（前後レコード移動＋引き継ぎ）・モデル比較

## 使い方

1. ☰設定で **計測デバイス**（デバイス名・マイク）を登録し、**シリーズ**（場所・対象・位置関係などの条件付き）を作る
2. 「音測定スタート」→ 数秒録音 → 判定バナー・類似度・品質警告
3. 測定直後にその場で **正常/異常ラベル**＋メモ（詳細でも可。← →で隣へ移動・前のラベルを引き継ぎ）。似た異常があれば「🔬 比較」で周波数プロファイルの一致/相違を確認
4. 「🧪 評価」で空間・正則化を選び「＋ このモデルを保存」
5. 以降の測定は保存モデルが自動判定。ラベルが増えたら「🧩 モデル」で再学習
6. 「JSONエクスポート」でバックアップ（音声は含まれません）

スマホではブラウザメニューの「ホーム画面に追加」で PWA として使えます。マイクは **localhost または HTTPS** でのみ動作します（ブラウザ仕様）。

## ローカルで試す

```bash
# リポジトリ直下で
python -m http.server 8000
# → http://localhost:8000
```

## データの持ち方

- 測定・埋め込み・ラベル・判定結果・**学習済みモデル** はすべて IndexedDB（端末内）
- シリーズ設定は localStorage
- 音声は端末外に一切送信しない
